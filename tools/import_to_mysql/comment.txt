სკრიპტის დანიშნულება
--------------------
ეს სკრიპტი გამოიყენება CSV ფაილში არსებული სეისმური მონაცემების
ავტომატურად ჩასატვირთად MySQL/MariaDB მონაცემთა ბაზის ცხრილში.

სკრიპტის ფუნქციონალი
--------------------
1) კითხულობს კონფიგურაციას .env ფაილიდან:
   - MySQL კავშირის პარამეტრები (ჰოსტი, ბაზა, მომხმარებელი, პაროლი)
   - სამიზნე ცხრილის სახელი
   - CSV ფაილის პარამეტრები (დელიმიტერი, ბრჭყალები, ჰედერის გამოტოვება)

2) ამოწმებს CSV ფაილის არსებობას და სისწორეს
   (ბილიკი განისაზღვრება სკრიპტის მდებარეობის მიხედვით).

3) უკავშირდება MySQL/MariaDB მონაცემთა ბაზას PyMySQL ბიბლიოთეკის გამოყენებით.

4) ახორციელებს მონაცემების ჩატვირთვას შემდეგი მეთოდით:
   - LOAD DATA LOCAL INFILE (მაღალი წარმადობა დიდი CSV ფაილებისთვის)
   - ავტომატურად ცდის სხვადასხვა ხაზის დამთავრების სიმბოლოებს
     (\n და \r\n) პლატფორმათაშორისი თავსებადობისთვის.

5) საჭიროების შემთხვევაში გამოტოვებს CSV ჰედერს
   (IGNORE N LINES პარამეტრის გამოყენებით).

6) უზრუნველყოფს ტრანზაქციის მართვას:
   - წარმატების შემთხვევაში commit
   - შეცდომის შემთხვევაში rollback

7) აწარმოებს დეტალურ ლოგირებას:
   - იმპორტის წარმატება ან წარუმატებლობა
   - გამოყენებული ხაზის დამთავრების სიმბოლო
   - შეცდომის დეტალები

რატომ არის ეს სკრიპტი საჭირო
----------------------------
ეს სკრიპტი ამარტივებს და აჩქარებს სეისმური მონაცემების
პირველადი ან პერიოდული ჩატვირთვის პროცესს CSV ფაილებიდან
მონაცემთა ბაზაში, განსაკუთრებით დიდი მოცულობის მონაცემების შემთხვევაში.

შენიშვნები
----------
- სკრიპტი განკუთვნილია ადგილობრივი CSV ფაილებისთვის
  (LOAD DATA LOCAL INFILE).
- აუცილებელია MySQL-ში ჩართული იყოს local_infile პარამეტრი.
- CSV ფაილის ფორმატი უნდა შეესაბამებოდეს ცხრილის სტრუქტურას.
- სკრიპტის გაშვება შესაძლებელია მრავალჯერ უსაფრთხოდ
  (დუბლიკატების დამუშავება დამოკიდებულია ცხრილის შეზღუდვებზე).
