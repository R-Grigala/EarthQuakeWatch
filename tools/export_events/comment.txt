სკრიპტის დანიშნულება
--------------------
ეს სკრიპტი იღებს ახლახან განახლებულ სეისმურ მოვლენებს
MySQL/MariaDB მონაცემთა ბაზიდან და აგზავნის მათ REST API
ენდპოინტზე JSON ფორმატში.

რას აკეთებს სკრიპტი
-------------------
1) კითხულობს კონფიგურაციას .env ფაილიდან:
   - MySQL კავშირის პარამეტრები (ჰოსტი, ბაზა, მომხმარებელი, პაროლი, ცხრილი)
   - API პარამეტრები (ბაზური URL, API გასაღები, timeout)

2) უკავშირდება MySQL/MariaDB მონაცემთა ბაზას PyMySQL ბიბლიოთეკის გამოყენებით.

3) ირჩევს მოვლენებს, რომლებიც აკმაყოფილებს შემდეგ პირობებს:
   - last_edit_time არის ბოლო 7 დღის ფარგლებში (UTC)
   - მაგნიტუდა (ml) მეტია 2.5-ზე
   სკრიპტი კითხულობს ძირითად ველებს, მათ შორის:
   event_id, origin_time, კოორდინატები, სიღრმე, რეგიონები და მაგნიტუდა.

4) გადადის მიღებულ მოვლენებზე და თითოეულს აგზავნის API-ში:
   - ენდპოინტი: {API_BASE_URL}/api/events
   - ავტორიზაცია: X-API-Key ჰედერი
   - Content-Type: application/json

5) გამოტოვებს მოვლენებს, რომელთაც არ აქვთ აუცილებელი კოორდინატები
   (latitude ან longitude).

6) აწარმოებს ლოგირებას:
   - წარმატებული გაგზავნა (HTTP 200/201)
   - დუბლიკატების დამუშავება (თუ API აბრუნებს „already exists“)
   - კლიენტის შეცდომები (4xx) და სერვერის შეცდომები (5xx)
   - ქსელის ან მოთხოვნის გამონაკლისები

რატომ არის ეს სკრიპტი საჭირო
----------------------------
ეს სკრიპტი ასრულებს მსუბუქ ETL/ინგესტაციის ფუნქციას და უზრუნველყოფს,
რომ API-ის მონაცემთა ბაზა სინქრონიზებული იყოს სეისმური კატალოგის
ცხრილში ბოლო ცვლილებებთან.

შენიშვნები
----------
- დროის ფანჯარა ამჟამად არის: ბოლო 7 დღე (UTC).
- მაგნიტუდის მინიმალური ზღვარი: ml > 2.5.
- სკრიპტის გაშვება შესაძლებელია მრავალჯერ უსაფრთხოდ
  (დუბლიკატები არ გამოიწვევს პრობლემას API-ის ქცევიდან გამომდინარე).
